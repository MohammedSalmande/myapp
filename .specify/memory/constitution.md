# Smart Task Manager Constitution

## Core Principles

### I. React + TypeScript Frontend
All frontend features must be implemented using React with TypeScript.
Components must be reusable, modular, and maintainable.
TailwindCSS should be used for styling.

---

### II. Spring Boot REST API
All backend functionality must be exposed through REST APIs using Spring Boot.
Controllers, Services, and Repositories must follow clean architecture principles.

---

### III. Scrum-Based Development
Development must follow Scrum principles.
Features should be implemented as small iterative tasks within sprint cycles.

---

### IV. Git & Version Control
Every meaningful change must be committed using Git.
Commit messages must clearly describe the implemented feature or fix.

---

### V. Dockerized Local Environment
The project must support local development using Docker Compose.
Frontend, Backend, and Database services should run independently in containers.

---

## Technology Stack

- Frontend: React / Next.js / TypeScript
- Backend: Java Spring Boot
- Database: MySQL
- Styling: TailwindCSS
- Containerization: Docker Compose
- Version Control: Git + GitHub

---

## Development Workflow

1. Define feature requirements
2. Create implementation tasks
3. Implement frontend/backend changes
4. Test functionality locally
5. Commit changes to Git
6. Push updates to GitHub

---

## Governance

This constitution defines the mandatory development standards for the Smart Task Manager project.

All contributors must follow:
- Clean architecture principles
- Consistent naming conventions
- Modular component design
- Clear Git commit history
