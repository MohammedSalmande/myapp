# Data Model: Smart Task Manager

## Entity: Task

Represents a single work item in the task manager.

### Fields

- `id`: `Long` / `number`
  - Primary key, auto-generated by the database.
- `title`: `String`
  - Required.
  - Short task summary.
- `description`: `String`
  - Optional details for the task.
- `urgent`: `Boolean`
  - Marks whether the task is urgent.
- `important`: `Boolean`
  - Marks whether the task is important.
- `status`: `TaskStatus`
  - Required enum with values: `TODO`, `IN_PROGRESS`, `DONE`.
- `dueDate`: `LocalDate` / `string`
  - Optional deadline for the task.
- `createdAt`: `Timestamp` / `string`
  - Automatically set when the task is created.
- `updatedAt`: `Timestamp` / `string`
  - Automatically updated when the task changes.

### Validation Rules

- `title` must be present and non-empty.
- `status` must be a valid enum value from `[TODO, IN_PROGRESS, DONE]`.
- `urgent` and `important` must be boolean values.
- `dueDate`, if present, must be a valid ISO date string.

## Derived Classification

The Eisenhower Matrix is derived from `urgent` and `important` flags:

- `urgent=true`, `important=true` → Do First
- `urgent=true`, `important=false` → Schedule
- `urgent=false`, `important=true` → Delegate
- `urgent=false`, `important=false` → Eliminate

## Relationships

- This MVP uses a single `Task` entity.
- No additional relationships are required for v1.

## Status Workflow

- `TODO` → initial state for new tasks.
- `IN_PROGRESS` → active work state.
- `DONE` → completed state.

## API Data Shape

### Task DTO

- `id`: number
- `title`: string
- `description`: string | null
- `urgent`: boolean
- `important`: boolean
- `status`: `TODO` | `IN_PROGRESS` | `DONE`
- `dueDate`: string | null
- `createdAt`: string
- `updatedAt`: string
